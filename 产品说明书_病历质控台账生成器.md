# 病历质控台账生成器 v2.0 — 产品说明书

**辽宁中医嘉和医院 · 医务科/医保办专用**

---

## 一、产品概述

病历质控台账生成器是一款面向医院病历质控场景的桌面端智能工具。系统对接 HIS 导出的 XPS 病历文件，通过 DeepSeek 大模型对病历进行六大维度、34 项细则的逐项智能检查，自动生成结构化缺陷台账，并支持绿色主题 Excel 一键导出。

### 核心价值

| 痛点 | 解决方案 |
|------|---------|
| 人工逐份查病历耗时 30-60 分钟/份 | AI 分析 15-30 秒/份，人工仅需复核 |
| 质控标准执行不一致 | 六维34项标准内置，每次检查全覆盖 |
| 缺陷描述不规范 | AI 润色引擎，口语转专业质控语言 |
| 台账整理费时费力 | 自动去重、合并、按人/按维度统计 |
| 历史缺陷无法追溯 | 本地历史记录，常用描述一键复用 |

## 二、功能模块

### 2.1 质控标准体系

系统内置辽宁中医嘉和医院病历质控六大维度、34 项检查细则及 5 项一票否决：

**一、核心时限（7项）**
入院记录 24h、首次病程 8h、主治查房 48h、主任查房 72h、连续 3 天病程、康复初始评定 72h、阶段小结 30 天

**二、康复专项（4项）**
标准化量表评定（Fugl-Meyer/Barthel/MMSE）、中期评定每 4 周、治疗记录单完整性、三单一致性

**三、病程记录质量（6项）**
首程要素完整、上级查房分析意见、病情变化记录、有创操作即时记录、患方签名、阶段小结非复制

**四、文书完整性（8项）**
一般项目无空项、三史无遗漏、体格检查完整、辅助检查有序、报告单签名、医师亲笔签名、首页完整、出院小结

**五、知情同意与告知（5项）**
授权委托书、特殊检查知情同意书、康复治疗风险告知书、各类告知书齐全、患方签名+日期

**六、书写规范（4项）**
无大段复制粘贴、双线划改规范、24 小时制记录、术语规范无矛盾

**一票否决项（5项）**
核心记录超时 24h 以上、康复初始评定完全缺失、知情同意书/告知书缺失或伪造签名、主要诊断与治疗严重不符、复制粘贴致重大错误（性别/左右侧）

### 2.2 自动化流水线

```
📑 XPS 源文件
    ↓ 一键提取
📄 提取文本 (_extracted/)
    ↓ AI 分析（DeepSeek 六维检查）
📊 分析结果 (_analysis/)
    ↓ 导入表单
📋 人工复核（修改/补充/确认责任人）
    ↓ 添加到台账
📦 台账列表（支持多份病历合并）
    ↓ 导出 Excel
📥 绿色主题 Excel 台账 + 汇总统计表
```

### 2.3 AI 智能分析

- 调用 DeepSeek API 逐份分析病历文本
- 自动提取患者姓名、住院号、病区、主管医师
- 按六维标准逐项检查，仅输出有缺陷的项（合格项不列）
- 一票否决项自动识别并标注
- 描述措辞专业客观（"需关注""建议完善"）
- 支持手动触发单份分析或批量分析

### 2.4 AI 文字润色

- 口语化缺陷描述 → 专业质控语言
- 支持单条润色和一键润色全部
- 润色历史本地存储（最近 15 条），点击即用
- 双通路：优先走灵枢后端 `/api/polish`，不可达时直连 DeepSeek

### 2.5 台账管理

- **智能合并**：同住院号多次添加自动合并，同检查项标注"补充"
- **在线编辑**：台账单元格双击即可修改，实时生效
- **逐条删除**：支持单条缺陷删除
- **一键清空**：支持批量清空台账
- **记录统计**：实时显示病历份数、缺陷总数

### 2.6 Excel 导出

- 绿色主题（医院品牌色系）
- 标题区（深绿背景白色粗体）→ 副标题（导出时间/缺陷数/病区）→ 表头 → 交替浅绿数据行
- 一票否决行自动标红加粗
- 底部汇总：病区统计、病历份数、缺陷总数、一票否决数
- 按维度统计明细
- **Sheet 2**：汇总统计表（按维度、按责任人分别统计）
- 文件名自动带日期

## 三、操作指南

### 3.1 启动程序

双击 `病历质控台账生成器.exe`，程序在后台静默启动（无黑窗），约 1-2 秒后自动打开浏览器进入台账页面。

如浏览器未自动打开，手动访问 `http://localhost:8081`。

### 3.2 首次配置

在页面顶部工具栏填入 **DeepSeek API Key**（格式：`sk-xxxx`），该 Key 会自动保存在本地浏览器中，下次打开无需重新输入。

API Key 获取：https://platform.deepseek.com → 注册 → API Keys → 创建

### 3.3 完整操作流程

#### 方式一：自动化面板（推荐）

1. 点击工具栏 **「🤖 导入AI结果」**，打开自动化面板
2. **📑 XPS 提取**：点击「刷新XPS列表」→ 选择病区文件 → 点击「提取」或「提取全部」
3. **🚀 一键分析**：点击「刷新待分析列表」→ 逐份分析或「分析全部待处理」
4. **📊 已有分析结果**：下拉选择已有分析 → 「导入选中」→ 缺陷自动填充到表单
5. **人工复核**：检查每条缺陷描述，修改不准确的内容，补充责任人姓名，确认一票否决项
6. **添加到台账**：点击「➕ 添加到台账」
7. **继续处理**：重复 2-6 步处理下一份病历
8. **导出**：全部完成后点击「📥 导出全部」

#### 方式二：全流程一键自动化

在自动化面板底部，点击 **「🚀 启动全流程」**，系统自动完成：
1. 提取所有未处理的 XPS 文件
2. 分析所有待处理的病历（调用 DeepSeek）
3. 全部导入台账

全程无需人工干预，完成后直接进入台账复核阶段。

#### 方式三：手动填写

直接在表单中：
1. 填写患者基本信息（日期、病区、床号、姓名、住院号、主管医师、检查人）
2. 在六维度的 textarea 中逐项描述缺陷（合格项留空）
3. 填写每条缺陷的责任人
4. 勾选涉及的一票否决项
5. 点击「✨」按钮可对单个描述进行 AI 润色
6. 点击「✨ 一键润色」可对所有已填项润色
7. 点击「➕ 添加到台账」
8. 全部完成后点击「📥 导出全部」

### 3.4 台账导出

导出后得到两份 Excel 工作表：

- **Sheet 1「质控台账明细」**：完整的缺陷清单，含日期/病区/床号/患者/住院号/主管医师/检查人/维度/检查项/缺陷描述/责任人
- **Sheet 2「汇总统计」**：按维度统计、按责任人统计，支持直接用于科室质控会议

## 四、技术架构

| 层级 | 技术选型 |
|------|---------|
| 后端框架 | Python Flask |
| 生产服务器 | waitress（纯 Python WSGI，Windows 原生支持） |
| AI 引擎 | DeepSeek Chat API（deepseek-chat 模型） |
| 前端 | 单页 HTML5 + Vanilla JS + ExcelJS |

| 数据存储 | 本地文件系统（JSON + TXT + XPS） |
| 支持平台 | Windows 10/11（无需 Python 环境） |

### 知识产权保护

- HTML 源码嵌入 EXE 二进制，不暴露为静态文件
- F12 开发者工具无法查看页面源码
- EXE 运行时无控制台窗口，后台静默运行
- 反编译保护：PyInstaller 打包混淆

### API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/` `/病历质控台账生成器.html` | GET | 台账主页面（内存渲染） |
| `/api/polish` | POST | AI 文字润色 |
| `/api/xps-list` | GET | 列出 XPS 文件及提取状态 |
| `/api/extract-xps` | POST | 提取指定 XPS 文本 |
| `/api/extract-all` | POST | 一键提取全部未处理 XPS |
| `/api/extracted-list` | GET | 列出已提取的文本文件 |
| `/api/run-analysis` | POST | 触发单份病历 AI 分析 |
| `/api/analysis-list` | GET | 列出已完成的分析结果 |
| `/api/analysis` | GET | 获取单个分析 JSON |

## 六、数据导入与维护

### 6.1 新增病历

将 HIS 导出的 XPS 文件放入对应病区文件夹（`data/病历质控测试组/A组/` 等），重启程序后即可在自动化面板中看到。

### 6.2 更新质控标准

质控标准（六维检查项）嵌入在服务端代码中。如需调整标准，请联系开发者更新后重新打包。

### 6.3 备份建议

定期备份 `data/病历质控测试组/` 目录，特别是 `_analysis/` 中的分析结果和导出的 Excel 台账。

## 七、常见问题

**Q: 双击 EXE 无反应？**
A: 检查 `config.json` 是否存在，检查 Windows 防火墙是否拦截了 8081 端口。可以尝试以管理员身份运行。

**Q: 浏览器没有自动打开？**
A: 手动访问 `http://localhost:8081`。如果页面无法加载，说明程序启动失败，检查 8081 端口是否被占用。

**Q: AI 分析返回"LLM not configured"？**
A: 未配置 DeepSeek API Key。在页面顶部工具栏填入有效的 API Key。

**Q: AI 分析失败？**
A: 检查网络连接和 API Key 有效性。DeepSeek 免费额度用完后需充值。单份病历约消耗 2000-4000 tokens。

**Q: 导入分析结果时匹配数为 0？**
A: 点击「刷新」重新从服务器加载分析列表。如果问题持续，请确认 `_analysis/` 目录下有 JSON 文件且格式正确。

**Q: 如何迁移到其他电脑？**
A: 将整个程序目录（含 EXE、config.json、data/）复制到目标电脑即可。新电脑可能需要安装 [VC++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)。

**Q: 如何卸载？**
A: 直接删除程序目录即可。程序不写注册表，不创建系统服务，无残留文件。
